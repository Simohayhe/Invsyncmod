InvSync Mod は、複数のサーバー間でプレイヤーのデータをリアルタイムに同期するためのFabric/Quilt向けサーバーサイドMODです。

BungeeCordやVelocityなどのプロキシ環境で、プレイヤーがサーバーを移動しても「同じインベントリ」「同じステータス」でプレイを継続できる環境を提供します。

🚀 主な機能
このMODを導入することで、以下のデータがサーバー間で同期されます：

インベントリ: メインインベントリ、防具スロット、オフハンド。

ステータス: 体力（HP）、空腹度、経験値（XP/レベル）。

エンダーチェスト: すべてのサーバーで共通のエンダーチェストを使用可能。

ポーション効果: 付与されているバフ・デバフ状態の維持。

🛠️ 導入方法
MODの配置: mods フォルダに本MODのjarファイルを入れます（サーバーサイドのみでOK）。

データベースの準備: データを共有するための MySQL または MariaDB サーバーを用意してください。

設定: 初回起動時に生成される config/invsync.json（または .conf）を開き、データベースの接続情報（ホスト、ポート、ユーザー名、パスワード）を入力します。

再起動: サーバーを再起動すれば同期が開始されます。

📋 前提条件
Fabric API / Quilt Standard Libraries

MySQL または MariaDB データベース

(推奨) サーバーネットワーク環境（Velocity, BungeeCord, Waterfall等）

⚠️ 注意事項
バックアップ推奨: 初めて導入する際は、必ずワールドデータとプレイヤーデータのバックアップを取ってください。

サーバーサイド専用: クライアント側に導入する必要はありません。
